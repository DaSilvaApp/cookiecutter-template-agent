# Architecture Diagrams

This folder contains the architecture diagrams for the project.  
Each diagram is created in Draw.io format and describes a different perspective of the system.

---

## 📌 Diagrams List

### 1. Functional Architecture

**File:** `functional-architecture.drawio`

**Goal:**  
Describes the high-level functionality of the system and how users interact with it.  
It shows the main features, user roles, and system boundaries.

---

### 2. Logical Architecture

**File:** `logical-architecture.drawio`

**Goal:**  
Shows the logical structure of the system including components, modules, and data flow.  
It focuses on how the system is organized internally, without detailing deployment.

---

### 3. Physical Architecture

**File:** `physical-architecture.drawio`

**Goal:**  
Represents the deployment view of the system.  
It shows servers, containers, cloud services, networking, and infrastructure.

---

### 4. Graph Implementation

**File:** `graph-implementation.drawio`

**Goal:**  
Shows how the system is implemented using agent patterns in a graph-based workflow. 
It can includes nodes, edges, and execution flow.  
This diagram is framework-agnostic and can be used with LangGraph or any graph library.

---

### 5. UML Sequence Diagram

**File:** `sequence-diagram.drawio`

**Goal:**  
Shows the detailed interaction flow between components over time.  
Includes method calls, responses, and the sequence of execution for a core use case.

---

## 📎 Notes

- All diagrams are stored in Draw.io format (`.drawio`).
- You can open and edit them using [diagrams.net](https://www.diagrams.net/).
- These diagrams are meant to provide clarity on both high-level and low-level system design.
